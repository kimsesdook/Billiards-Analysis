# 프로젝트 기획서: AlgoDelivery (스마트 물류/배달 라우팅 시스템)

## 1. 프로젝트 개요
실시간 물리적 제약(거리, 트럭 적재량, 주문 쇄도)이 있는 물류/배달 비즈니스를 가정하여, 시스템 효율성을 극대화하기 위해 '백엔드 서버 내 메모리에 직접 구현한 자료구조와 알고리즘'을 핵심 비즈니스 로직(Core Logic)으로 활용하는 API 서비스 포트폴리오.
코딩테스트의 S급~B급 핵심 알고리즘을 100% 실무형 로직으로 완전히 치환하여 포트폴리오의 변별력을 높임.

---

## 2. 핵심 기능 및 알고리즘 매핑 (코테 완벽 대비)

### 2-1. 실시간 대기열 제어 (자료구조 : Queue & Priority Queue)
* **문제:** 수천 건의 배달 요청이 몰릴 때 기사에게 효율적으로 배차
* **해결 (구현):**
  - 일반 주문: `CustomQueue`(FIFO)에 담아 순차 처리.
  - VIP/로켓 주문: `CustomPriorityQueue`(Min Heap)에 담아 큐를 새치기하여 최우선 배차.
* **UI 연출:** 통합 대시보드 화면. 네모난 박스들이 일반 큐에 차곡차곡 쌓이고, VIP 상자는 힙 알고리즘에 의해 맨 앞으로 꽂혔다가 사라지는 배차 애니메이션.

### 2-2. 초고속 주소/운송장 검색 머신 (자료구조: Trie & Hash Map)
* **문제:** DB에서 LIKE 검색을 하면 시스템 병목 발생
* **해결 (구현):**
  - 서버 메모리 내에 `CustomTrie`(트라이) 자료구조를 구축하여 O(문자열 길이) 속도로 주소 자동완성.
  - 임의의 난수 형태인 운송장 번호는 `CustomHashMap`으로 O(1) 초고속 조회.
* **UI 연출:** 네이버 스타일의 대형 검색창. 텍스트를 입력할 때마다 0.01초 만에 하단에 자동완성 리스트가 드롭다운됨.

### 2-3. 최단 경로 네비게이션 API (알고리즘: Graph & Dijkstra)
* **문제:** 복잡한 도로망과 변수(교통 체증, 공사) 속에서 가장 빠른 배송 경로 도출
* **해결 (구현):**
  - 교차로(Node), 도로(Edge), 교통체증(Weight) Graph 생성. 교통 체증 변동 시 가중치 즉시 변경.
  - `Dijkstra`(다익스트라) 알고리즘 API를 호출하여 최단경로 좌표 배열(JSON) 반환.
* **UI 연출:** 카카오/네이버 지도 연동. 맵에서 '공사 중' 버튼을 눌러 특정 간선의 가중치를 무한대로 만든 후 [최단 경로 계산] 클릭 시, 우회하는 빨간 선이 스르륵 그어짐.

### 2-4. 트럭 적재량 최적화 시스템 (알고리즘: 동적 계획법 - Knapsack dp)
* **문제:** 제한된 트럭의 하중 내에서 배송 수익을 가장 극대화할 수 있는 택배 조합 찾기
* **해결 (구현):**
  - 배낭 문제(0-1 Knapsack Algorithm) DP 로직 적용.
  - DP 캐싱으로 빠른 연산 수행 후 최적화된 택배 번호 리스트 추출.
* **UI 연출:** 트럭 테트리스 시뮬레이션. 남은 공간 대비 효율적인 택배 상자들이 자동으로 트럭 안으로 이동하는 직관적인 화면.

## 3. 개발 환경 및 아키텍처 (REST API 구조)
* **프론트엔드와 백엔드 완벽 분리** (알고리즘 계산의 전문성 어필)
* **Backend:** Java 17, Spring Boot 3.x, JPA(Hibernate) - JSON만 반환하는 REST API 서버
* **Database:** MySQL (로그 기록용) + 서버-사이드 In-Memory 연산(자료구조 빡구현)
* **Frontend:** 순수 HTML + Vanilla JS (또는 React/Vue) - 단순 시각화 및 API fetch 용도

## 4. 백엔드 패키지 구조 설계 (예상도)
```text
src/main/java/com/yourname/algodelivery
├── core/                  # ★ 이 포트폴리오의 심장 (순수 자바 알고리즘 엔진)
│   ├── datastructure/     # CustomQueue, CustomHashMap 등
│   └── algorithm/         # DijkstraRouter, KnapsackOptimizer 등
│
├── controller/            # REST API 엔드포인트 (프론트엔드와 JSON 통신)
├── dto/                   # 데이터 교환 객체 (요청 Request / 응답 Response)
├── service/               # 비즈니스 로직 및 코어 엔진 호출, 트랜잭션 관리
├── repository/            # DB 접근 인터페이스 (Spring Data JPA)
└── entity/                # DB 테이블과 1:1 매핑되는 JPA Entity 클래스
```
